Hive之路1-初识Hive

Hive简介

什么是Hive
  1. Hive是由Facebook实现并且开源
  2. 基于Hadoop的开源的数据仓工具，完成ETL（extract transform load），报表及数据分析等
  3. Hive能够将结构化的数据映射称为一张表
  4. Hive提供HQL（Hive SQL）的查询功能
  5. 底层的数据是存储在HDFS之上，使用于离线的批量数据计算
  6. Hive的本质是将SQL语句转换成MapReduce任务进行运行
  7. LLAP（Live Long And Process），使Hive实现内存计算

Hive 依赖于 HDFS 存储数据，Hive 将 HQL 转换成 MapReduce 执行，所以说 Hive 是基于 Hadoop 的一个数据仓库工具，实质就是一款基于 HDFS 的MapReduce 计算框架，对存储在 HDFS 中的数据进行分析和管理

为什么使用Hive
  1. 更友好的接口：操作接口采用类 SQL 的语法，提供快速开发的能力
  2. 更低的学习成本：避免了写 MapReduce，减少开发人员的学习成本
  3. 更好的扩展性：可自由扩展集群规模而无需重启服务，还支持用户自定义函数

Hive 内置数据格式

  • 逗号和Tab字段分隔的文本文件
  • Apache Parquet文件
  • Apache ORC文件
  • 其他格式

Hive内置函数

  1. 标准的SQL函数
  2. 用户自定义函数UDF
  3. 用户自定义聚合函数UDAF
  4. 用户自定义表函数UDTF

Hive特点

优点

  1. 可扩展性、横向扩展

    Hive可以自由地扩展集群的规模，一般情况下不需要重启服务；

  2. 延展性

    Hive支持自定义函数，用户能够根据自己的要求来实现自己的函数

  3. 良好的容错性

    即使部分节点出现问题，SQL语句仍可以正常运行

缺点

  1. 不支持记录级别的增删改操作，但是用户可以通过查询生成新的表格
  2. Hive的查询延时很严重，因为 MapReduce Job 的启动过程消耗很长时间，所以不能用在交互查询系统中
  3. Hive不支持事务功能，没有增删改，主要是用来做OLAP（联机分析处理），而不是OLTP（联机事务处理）

用户接口

Hive中有3种访问方式

  1. CLI ：command line interface，命令行接口，采用交互式使用Hive命令和Hive进行交互，最常用
  2. JDBC/ODBC：客户端中执行，需要指定相应的运行节点，并在节点上启动Hive Server
  3. Web UI：通过浏览器来访问
元数据存储系统

Metastore元数据就是存储在Hive中的数据的描述信息，包含：

  • 表的名字
  • 表的列和分区及属性
  • 表的属性（内部表和外部表）
  • 表的数据所在的目录等

Metastore元数据默认是存储在关系型的Derby数据库中，管理极度不方便，且不适合多用户操作

解决方案：采用自建的MySQL数据库。Hive和MySQL之间通过MetaStore服务进行交互。

Driver

Driver中包含了各种驱动器Driver，编译器Compiler，优化器Optimizer，执行器等；

Driver组件完成 HQL 查询语句从词法分析，语法分析，编译，优化，以及生成逻辑执行 计划的生成。最终的查询计划保存在HDFS中，并且由mapreduce调用执行，转成mapreduce的作业。

Hive的核心是驱动引擎， 驱动引擎由四部分组成：

  • 解释器：解释器的作用是将 HiveSQL 语句转换为抽象语法树（AST）

  • 编译器：编译器是将语法树编译为逻辑执行计划

  • 优化器：优化器是对逻辑执行计划进行优化

  • 执行器：执行器是调用底层的运行框架执行逻辑执行计划

Hive元数据连接模式

  • 单用户模式：默认是存储在Derby中，只支持一个会话连接，适合简单的测试，实际生产环境中不适用
  • 多用户模式：为了支持多会话模式，需要一个独立的元数据库，一般采用mysql
  • 远程模式

Hive执行流程

  1. Hive SQL通过命令行或者客户端进行提交
  2. 经过编译器，运用元数据进行检测和语法分析，生成一个逻辑方案
  3. 通过优化处理，产生一个MapReduce作业进行处理

Hive数据组织

Hive中的存储结构包含数据库、表、视图、分和表数据等。数据库，表，分区等等都对 应 HDFS 上的一个目录。表数据对应 HDFS 对应目录下的文件。

  1. Hive 的中数据都是存储在HDFS中，没有专门的数据存储格式

  2. 在创建表的时候需要指定数据中的行列分隔符：

    • 默认列分隔符：控制符Ctrl + A

    • 默认行分隔符：换行符\n

  3. Hive中包含的数据模型

    • database：在 HDFS 中表现为${hive.metastore.warehouse.dir}目录下一个文件夹

    • table：在 HDFS 中表现所属 database 目录下一个文件夹

    • external table：与 table 类似，不过其数据存放位置可以指定任意 HDFS 目录路径

    • partition：在 HDFS 中表现为 table 目录下的子目录

    • bucket：在 HDFS 中表现为同一个表目录或者分区目录下根据某个字段的值进行 hash 散 列之后的多个文件

    • view：与传统数据库类似，只读，基于基本表创建

  4. Hive中的元数据存储在RDMBS中，除元数据之外的其他数据都是基于HDFS存

Hive中的表

Hive中的表分为：内部表、外部表、分区表和桶表

内部表和外部表的使用选择：

  1. 多数情况下，二者区别不明显；如果所有的数据都在Hive中进行处理，选择内部表；如果是Hive和其他工具的结合使用，选择外部表
  2. 使用外部表访问存储在HDFS上的数据，通过Hive转换数据并且存到内部表中

分区表和分桶表：

  1. Hive数据表根据字段进行分区操作，细化数据管理，加快查询数据
  2. 表和分区可以进一步划分成Buckets
  3. 分区表是手动添加分区的，Hive是读模式，对添加进分区的数据不做模式校验
  4. 分桶表中的数据是按照某些字段进行hash散列行成的文件，数据的准确性更高
